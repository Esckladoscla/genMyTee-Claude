# Product Positioning

## Core product idea
The flagship product is customizable apparel (t-shirts, hoodies, sweatshirts, etc.) where the customer describes the design they want in natural language and receives a generated visual concept applied to the garment.

## Positioning
The product should be marketed primarily as:
- personalized
- expressive
- made from the customer's idea
- emotionally meaningful
- visually unique for the customer

The product should not be marketed primarily as “AI-generated clothing”.
AI is the enabling technology, not the emotional value proposition.

## Messaging principles
- Lead with personalization and identity
- Sell the result, not the tool
- Keep AI as secondary explanatory context when useful
- Avoid cold or overly technical language in customer-facing copy
- Keep messaging in Spanish for the user-facing storefront unless otherwise requested

## Recommended messaging direction
Preferred ideas:
- “Tu imaginación, hecha prenda”
- “Diseñada por ti”
- “Convierte tu idea en una prenda que puedas llevar”
- “Crea algo tuyo”

## Avoid
- Overly technical “AI-first” headlines
- Generic “generated by AI” framing as the main selling point
- Claims like “única en el mundo” unless they are truly defensible

## Product UX implications
The storefront should make it easy for the customer to:
1. choose garment type
2. choose color
3. write a prompt naturally
4. get inspiration from examples
5. preview the result
6. choose size and quantity
7. purchase with confidence

The customization experience is a core part of the product, not an extra feature.

## Reference documents

When working on this repository, also read and take into account these project documents:

- @docs/product-positioning.md  
  Source of truth for product positioning, messaging direction, and how the customizable apparel experience should be presented to customers.

- @references/frontend-reference.html  
  Visual and UX reference for the desired frontend direction.  
  Treat this as a reference for layout, flow, hierarchy, and product presentation, not necessarily as final production code.

If code in the repository reflects the current implementation but these documents reflect the intended direction, distinguish clearly between:
- current state,
- target direction,
- and recommended migration path.

## Frontend direction

The frontend is still under active construction and may change significantly.

Claude should assume that:
- the current frontend is not necessarily the final one;
- frontend redesign is an active priority;
- the desired frontend direction should follow the reference HTML and the product positioning brief;
- the customization flow is a core product experience, not a secondary feature.

When working on frontend-related tasks:
- use `references/frontend-reference.html` as the visual/UX direction;
- use `docs/product-positioning.md` as the messaging and product framing direction;
- do not blindly preserve Shopify theme patterns if they conflict with the intended product evolution;
- explain gaps between the current frontend and the desired direction before proposing major changes.

## Conflict handling

If there is a conflict between:
- the current implementation,
- the product positioning brief,
- and the frontend reference,

do not silently choose one.
Instead:
1. identify the conflict clearly,
2. explain what belongs to current state vs intended direction,
3. recommend the most sensible next step.
